Chiến lược: Đẩy logic xử lý xuống Database để Frontend tập trung làm UI/Trải nghiệm

Database & Backend Logic Nguyễn Quang Nhựt
#1 Cảnh báo Chat tự động Logic nặng
  • Viết Database Trigger (hoặc Supabase Edge Function) chặn ở bảng messages.
  • Logic: Hễ có người dùng INSERT tin nhắn chứa chữ "chuyển khoản", "cọc"... thì DB tự động sinh thêm 1 tin nhắn cảnh báo (System message màu đỏ) chèn luôn vào khung chat đó. (Để Minh không phải code logic soi tin nhắn trên Frontend).
#2 Xử lý Report & Khóa tài khoản Logic nặng
  • Tạo bảng user_reports để lưu trữ lượt báo cáo.
  • Thêm cột is_locked vào bảng profiles/roommates.
  • Viết Database Trigger đếm số lượng report. Nếu tài khoản bị > 3 người khác nhau report → DB tự động update is_locked = true. (Để Minh gọi ra hiển thị luôn bên Frontend mà không cần ngồi tính toán).
#5 Tự động hết hạn bài đăng Logic nặng
  • Cài đặt Cron Job (pg_cron) chạy ngầm trên Supabase mỗi ngày.
  • Logic: Tự động quét bảng rooms / roommates, hễ bài nào có created_at quá 30 ngày thì tự động update status = 'expired'.
⚠️ Sync với Minh (Frontend): Minh đã làm nút "Gia hạn bài đăng" ở trang Lịch Sử.
Khi user bấm nút đó, frontend gọi Supabase update created_at = NOW() về ngày hôm nay → đồng hồ 30 ngày reset lại từ đầu.
👉 Nhựt cần đảm bảo cron job dùng đúng field created_at để tính ngày hết hạn (không dùng field khác).
👉 Sau khi cron update status = 'expired', bài sẽ bị ẩn khỏi danh sách tìm kiếm (Frontend đã filter theo status).
#5b Gửi Email thông báo sắp hết hạn Logic nặng
  • Dùng Supabase Edge Function kết hợp cron job để tự động gửi email nhắc nhở.
  • Điều kiện: Bài đăng còn đúng 2 ngày nữa là hết hạn → gửi email tới địa chỉ email của chủ bài đăng.
  • Nội dung email mẫu: "Bài đăng của bạn sắp hết hạn hiển thị. Nếu vẫn đang tìm roommate, hãy đăng nhập và bấm Gia hạn để tiếp tục hiển thị."
  • Có thể dùng service gửi mail miễn phí như Resend hoặc SendGrid tích hợp với Supabase Edge Function.
⚠️ Lưu ý: Frontend (Minh) đã làm sẵn nút "Gia hạn bài đăng" trong trang Lịch Sử Thỏa Thuận. Email chỉ cần dẫn user vào trang đó là xong, không cần làm thêm giao diện mới.
#4 & #6 Schema Hóa đơn & Dọn Data giả
  • Bảng rooms: Thêm cột bill_image (lưu ảnh hóa đơn) và đổi mặc định status = 'pending'. Tạo bucket Storage cho bill.
  • Dọn dẹp (Xóa) sạch toàn bộ dữ liệu mẫu do AI/Clone tạo ra bằng script SQL.
⚠️ Sync với Minh (Frontend): Minh đã làm form upload ảnh hóa đơn và lưu URL vào field utilityImage trong bảng rooms.
👉 Nhựt cần tạo bucket Storage tên "utility-bills" (hoặc dùng chung bucket ảnh hiện tại) và cấp quyền public read cho bucket đó để ảnh hiển thị được trong Admin Dashboard.
#8 Tạo bảng admin_roles BẮT BUỘC
  • Chạy SQL sau trong Supabase SQL Editor để tạo bảng phân quyền Admin:
CREATE TABLE admin_roles (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid NOT NULL UNIQUE,
  created_at timestamptz DEFAULT now()
);
  • Cấp quyền RLS cho bảng: Admin Dashboard đọc được, user thường không đọc được.
⚠️ Tại sao cần bảng này: Minh đã làm tính năng Phân quyền Admin trong Dashboard — Admin có thể bấm nút ⭐ để cấp/thu hồi quyền Admin cho bất kỳ user nào mà không cần đụng vào code hay Vercel.
👉 Khi Nhựt tạo xong bảng này, tính năng sẽ hoạt động ngay lập tức. Nếu chưa có bảng, bấm nút sẽ báo lỗi.
#1 & #3 Setup Trigger Chat Hệ thống
  • FE đã làm xong UI tin nhắn cảnh báo. Nhựt cần setup Trigger trên Supabase để tự động quét từ khóa "chuyển khoản", "cọc" và bắn tin nhắn vào bảng messages với flag is_system = true.
#4 Thêm DB: Lý do từ chối (Reject)
  • Bảng roommates & rooms: Thêm cột rejectReason (kiểu Text). FE đã làm xong nút Reject và luồng nhập lý do bên Admin Dashboard, chỉ chờ có cột này trong DB để lưu.
#10 Setup DB: Lưu trữ Report (Báo cáo)
  • FE đã làm xong UI Report, chặn spam và hiện đang gửi tạm data report vào bảng messages.
  • Nhựt cần xem xét tạo bảng reports riêng trên Supabase (có các cột: target_id, reason, source...) và sửa lại API gửi report bên FE cho chuẩn.
Frontend & User Interface Lê Quang Minh
#7 Xuất PDF Thỏa thuận Chính ✅ Xong
  • Sử dụng thư viện html2pdf.js.
  • Thiết kế template giao diện (ẩn) theo form A4 và làm nút Tải xuống PDF trong khung Thỏa thuận ở chung. Đảm bảo in ra đẹp và chuyên nghiệp.
#4 Form Đăng tin & Luồng Kiểm duyệt Chính ✅ Xong
  • Cập nhật form Đăng phòng: Thêm input tải ảnh Hóa đơn (upload lên Supabase Storage, lưu URL vào field utilityImage) và Checkbox cam kết bắt buộc tích trước khi đăng.
  • Xây dựng trang Admin Dashboard: Hiện danh sách bài đang pending. Hiển thị ảnh Hóa đơn + Nút Check ảnh mạng (Google Lens) + Nút Approve / Reject.
#2 & #5 Giao diện Report & Cảnh báo hạn ✅ Xong
  • Gắn nút Report vào từng card hồ sơ (Tìm Bạn) và card phòng (Phòng Trọ). Báo cáo lưu vào bảng user_reports trong Supabase. Admin xem được trong Dashboard.
  • Tại tab "Bài đăng của tôi" trong Lịch Sử: Tính ngày hết hạn từ created_at + 30 ngày. Hiển thị badge màu (🟢 còn nhiều / 🟡 sắp hết ≤3 ngày / 🔴 đã hết hạn) + Nút "Gia hạn bài đăng".
  • Khi bấm Gia hạn: Frontend update created_at = NOW() trên Supabase → reset đồng hồ 30 ngày.
#1 & #3 Render UI Chat & Bộ lọc ✅ Xong
  • Trang chủ tìm kiếm: Lọc bỏ (Filter) các hồ sơ có status = 'Đã có phòng' khỏi danh sách Tìm Bạn.
  • Khung Chat: Tin nhắn nào có is_system = true sẽ tự động hiển thị style riêng (nền đỏ nhạt, icon ⚠️, chữ đậm) — chờ Nhựt setup Trigger bắn tin nhắn hệ thống vào bảng messages với flag is_system = true.
#9 Ẩn bài viết cũ
  • Viết logic tự động lọc và ẩn các bài đăng đã quá 30 ngày (dựa vào createdAt) để tránh loãng data cũ.
#10 Tự động tải PDF & Chặn Spam Report
  • Thỏa thuận ở chung: Cài đặt logic tự động tạo và tải xuống file PDF ngay sau khi người dùng bấm Ký Xác Nhận.
  • Báo cáo (Report): Thêm kiểm tra ở Frontend để mỗi tài khoản chỉ được phép báo cáo (report) một nội dung duy nhất 1 lần. Hiện cảnh báo nếu cố tình spam report.
